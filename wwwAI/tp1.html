Les tests : prendre des décisions

Apprends à faire des comparaisons dans ton code !

C'est quoi un test en programmation ?

Dans la vie de tous les jours, tu prends des décisions tout le temps : "S'il pleut, je prends mon parapluie. Sinon, je n'en prends pas." En programmation, c'est exactement pareil ! On appelle ça un test ou une condition.

La syntaxe du test : if

En C#, pour faire un test, on utilise le mot-clé if (qui veut dire "si" en anglais).

Tu peux comparer des valeurs avec :

Symbole Signification Exemple
== Est égal à age == 10
!= Est différent de age != 10
> Est supérieur à age > 10
< Est inférieur à age < 10
>= Supérieur ou égal age >= 10
<= Inférieur ou égal age <= 10

Premier exemple 

var agePere = 40;
var ageMere = 35;

if (agePere > ageMere)
    Console.WriteLine("Mon père est plus vieux que ma mère.");

Ce code compare l'âge du père et de la mère. Si le père est plus vieux, le message s'affiche. Sinon, il ne se passe rien.

Les blocs de code

En C#, on utilise des accolades { } pour regrouper plusieurs instructions ensemble. C'est ce qu'on appelle un bloc de code. 

if (condition)
{
    instruction 1;
    instruction 2;
    instruction 3;
}

À retenir :

  • La condition est toujours entre parenthèses ( )
  • Si tu n'as qu'une seule instruction, les accolades sont optionnelles
  • Si tu as plusieurs instructions, tu dois utiliser les accolades

Gérer tous les cas

Que se passe-t-il si ton père et ta mère ont le même âge ? Il faut prévoir ce cas également ! 

var agePere = 40;
var ageMere = 40; // Même âge

if (agePere > ageMere)
{
    Console.WriteLine("Mon père est plus vieux que ma mère.");
}
if (ageMere > agePere)
{
    Console.WriteLine("Ma mère est plus vieille que mon père.");
}
Console.WriteLine("Ce programme est terminé.");

Réfléchis :

Dans cet exemple, aucun des deux messages de comparaison ne s'affichera, car les deux âges sont identiques. Seul le dernier message "Ce programme est terminé." apparaîtra. Tu vois le problème ? On n'a pas prévu le cas où les deux âges sont égaux !

C'est à ton tour — TP 01

Crée un nouveau projet C# (appelle-le TP01) et recopie le code ci-dessus.

Voici ce que tu dois faire :

  • 1
    Ajoute un test qui affiche un message quand agePere == ageMere (quand ils ont le même âge)
  • 2
    Essaie d'écrire if (agePere = ageMere) avec un seul égal au lieu de deux. Que se passe-t-il ? Pourquoi ?

Erreur classique à éviter !

= (un seul égal) = affecter une valeur à une variable
== (deux égaux) = comparer deux valeurs

Si tu confonds les deux, le compilateur sera très mécontent !
